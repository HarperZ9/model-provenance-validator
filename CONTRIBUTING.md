# Contributing

Thanks for improving `model-provenance-validator`.

## Development

1. Create a branch from `main`.
2. Install test dependencies with `python -m pip install -e ".[test]"`.
3. Run `python -m pytest`.
4. Keep schemas small, explicit, and covered by fixtures.

## Security

Do not submit private model receipts, provider credentials, private benchmarks,
customer prompts, or proprietary evaluations. Synthetic fixtures are preferred.
